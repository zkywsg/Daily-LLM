# Contributing to Daily-DeepLearning

贡献指南 / Contributing Guide

## How to Contribute / 如何贡献

Thank you for considering contributing to Daily-DeepLearning! We welcome all contributions that improve the project's educational value.

欢迎您为 Daily-DeepLearning 做出贡献！我们欢迎所有能提升项目教育价值的贡献。

### Steps / 步骤

1. Fork the repository / Fork 本仓库
2. Create a feature branch / 创建功能分支 (`git checkout -b feature/amazing-feature`)
3. Make your changes / 进行修改
4. Commit your changes / 提交修改 (`git commit -m 'Add some amazing feature'`)
5. Push to the branch / 推送到分支 (`git push origin feature/amazing-feature`)
6. Open a Pull Request / 开启 Pull Request

## Code Style / 代码规范

### Python

- Follow PEP 8 style guide / 遵循 PEP 8 风格指南
- Use meaningful variable names / 使用有意义的变量名
- Add docstrings to functions / 为函数添加文档字符串
- Maximum line length: 88 characters (black default) / 最大行长度：88 字符（black 默认值）

### Markdown

- Use consistent heading levels / 使用一致的标题层级（#, ##, ###）
- Specify language for code blocks / 为代码块指定语言（\`\`\`python, \`\`\`bash）
- Add alt text for images / 为图片添加替代文本

## Commit Message Convention / 提交信息规范

We follow the conventional commits format / 我们遵循 conventional commits 格式。

### Format / 格式

`type(scope): description`

### Types / 类型

- `feat`: A new feature / 新功能
- `fix`: A bug fix / 修复 bug
- `docs`: Documentation changes / 文档更改
- `style`: Code style changes (formatting, etc.) / 代码样式更改（格式化等）
- `refactor`: Code refactoring / 代码重构
- `test`: Adding or updating tests / 添加或更新测试
- `chore`: Maintenance tasks / 维护任务

### Examples / 示例

- `fix(readme): correct spelling error` - Fix README typos / 修复 README 拼写错误
- `feat(transformer): add attention mechanism` - Add attention to Transformer / 为 Transformer 添加注意力机制
- `docs(cnn): update architecture diagram` - Update CNN diagram / 更新 CNN 架构图

## Pull Request Process / PR 流程

1. Small PRs are easier to review / 小 PR 更容易审查
2. Update documentation for new features / 为新功能更新文档
3. Add tests when applicable / 适用时添加测试
4. Ensure CI passes (when CI is added) / 确保 CI 通过（当 CI 添加后）
5. Respond to review comments promptly / 及时回应审查评论

## Code of Conduct / 行为准则

Be respectful and constructive. We're here to learn together.

请保持尊重和建设性态度。我们在这里共同学习。

---

Thank you for contributing! / 感谢您的贡献！
