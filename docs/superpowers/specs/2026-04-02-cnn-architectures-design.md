## 签名元素

### 这个问题从哪来

中文版本建议开头：

> 2012 年之前，视觉识别系统大量依赖 SIFT、HOG 等手工特征，图像先被设计成特征，再交给分类器。即便使用多层感知机，把图像直接压平成向量也会带来参数爆炸，并破坏像素的空间邻域关系。AlexNet 的突破不只是“更深”，而是第一次大规模证明：把局部性和参数共享写进网络结构，才能让模型真正利用图像这种数据的形状。

英文版本对应强调同一含义，不逐句直译，保留术语准确性：

> Before AlexNet, visual recognition relied heavily on hand-crafted features such as SIFT and HOG. Flattening an image into a vector made MLPs expensive and structurally blind to spatial locality. The real breakthrough of CNNs was not depth alone, but the decision to encode locality and parameter sharing directly into the architecture.

### 你要记住（全模块 ≤ 3 次）

1. 直觉部分后:
   `CNN 的本质不是“层更多”，而是“把空间结构写进模型假设里”。`
2. 残差部分后:
   `ResNet 真正解决的不是“表达能力不足”，而是“信息和梯度跨深层流动困难”。`
3. 演进笔记末:
   `CNN 强在局部归纳偏置，限制也同样来自这种固定偏置。`

### 演进笔记

中文结尾建议：

> **这一技术的遗产**：CNN 把“局部性 + 参数共享”发挥到极致，让视觉模型第一次能稳定、系统地从像素中学习层级特征。它在数据较少、空间先验明确时依然极有优势。但这种固定局部窗口也意味着：模型更擅长从近邻累积全局信息，而不擅长一开始就灵活建模远程依赖。
>
> 这正是后续视觉模型逐步引入注意力机制、并最终走向 ViT 的原因。
>
> → 后续汇流：Transformer / 注意力架构

英文版本应表达同一论点，并与后续 Transformer 章节的术语保持一致。

## 章节结构

遵循 [`STYLE.md`](../../../STYLE.md) 模板，但重写为更强的问题链。

### 1. `## 这个问题从哪来`

- 先讲图像的空间先验，而不是先讲比赛成绩
- 解释“压平图像 + 全连接”为什么同时导致：
  - 参数量爆炸
  - 空间邻域关系丢失
  - 对平移等局部模式缺乏自然共享机制
- 再引出 AlexNet 作为第一代成功工程回答，而不是单纯作为历史节点

### 2. `## 学习目标`

建议统一为三问：

1. 卷积的局部连接、参数共享、层级特征分别解决了什么问题？
2. 为什么 VGG、GoogLeNet、ResNet、DenseNet、SE-Net 会按这样的顺序出现？
3. CNN 为什么在视觉任务中强大，又为什么最终要与注意力机制汇流？

### 3. `## 1. 直觉`

采用三步递进：

1. 图像的结构不是“任意维度集合”，而是二维局部相关信号
2. 全连接层对图像既浪费参数，又无视空间结构
3. 卷积核像一组可复用的小探测器，在全图上扫描同类模式

保留“探照灯”式类比，但要把重点从形象比喻拉回结构先验。

### 4. `## 2. 机制`

收紧为四节，避免百科式铺开：

#### 2.1 卷积与特征图

- 二维卷积公式
- kernel / stride / padding / channel 的语义
- 特征图不是“压缩图像”，而是“模式响应图”

#### 2.2 感受野与下采样

- 为什么现代 CNN 倾向堆叠多个 3×3，而不是直接用大核
- pooling / stride 的收益与代价
- “有效感受野”与“理论感受野”区别只点到为止，不展开成独立专题

#### 2.3 残差连接

- 深层网络的退化不是简单的过拟合问题
- shortcut 如何改善信息流和梯度流
- 为什么 identity mapping 是 ResNet 的关键直觉

#### 2.4 从局部到全局的限制

- CNN 可以通过堆层、扩感受野和多尺度设计逼近全局信息
- 但信息融合路径仍是逐层传播为主
- 这一局限为后续注意力机制埋下过渡点

### 5. `## 3. 架构演进`

这是全章核心，采用“上一代瓶颈 -> 新架构回应”的统一写法。

#### AlexNet

- 回应的问题: 手工特征难以扩展，深度网络训练和算力尚未形成稳定闭环
- 保留要点: 深度卷积、ReLU、GPU、Dropout
- 重点写法: 它是“把 CNN 跑通”的工程里程碑，不是理论终点

#### VGG

- 回应的问题: AlexNet 设计仍带经验性，结构不规整
- 要点: 用统一 3×3 堆叠建立“深而整齐”的范式
- 代价: 参数和计算量迅速膨胀

#### GoogLeNet

- 回应的问题: VGG 太重
- 要点: Inception 多尺度并行 + 1×1 降维
- 读者 takeaway: 它体现“同层多尺度特征提取”的设计思想

#### ResNet

- 回应的问题: 深度继续增加时训练退化
- 要点: shortcut、残差学习、identity path
- 这是整章最需要讲透的一代

#### DenseNet / SE-Net

- DenseNet 回应的问题: 特征复用仍不充分
- SE-Net 回应的问题: 通道被一视同仁，缺少动态重标定
- 这里只作为主线后段的“两个方向上的精修”，不展开成与 ResNet 等长的篇幅

### 6. `## 4. 工程陷阱`

按优先级收敛为 4 条，且每条都服务本章主线：

1. **shortcut 未对齐** -> 通道数或 stride 不匹配，残差相加直接报错
2. **过早下采样** -> 计算省了，但高频细节和小目标信息被提前抹掉
3. **只堆层数，不看感受野设计** -> 网络很深，但看不到足够大上下文
4. **BN / activation / residual 顺序混乱** -> 训练不稳定，性能与论文实现偏差明显

不再列过多泛化训练建议，避免与 `01-Visual-Intelligence/training` 重叠。
