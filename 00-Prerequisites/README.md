# Phase 01: Foundations

> 这一阶段解决的不是“怎么调用模型”，而是“怎么把真实业务问题抽象成可建模、可评估、可迭代的问题”。

## 为什么这一阶段值得先读

很多人一上来就学 Transformer、RAG 或 Agent，结果在真正落地时反而卡在更基础的问题上：目标怎么定义、指标怎么选、数据为什么会泄漏、模型为什么离线好看但线上失效。
Phase 01 的目标，就是先把这些底层判断力建立起来。

这一阶段会用一个贯穿案例带你走完整个思考过程：一家消费金融平台希望预测用户未来 90 天是否会违约。你会先用传统机器学习建立可信基线，再进入深度学习，理解神经网络为什么强、也理解它为什么并不总是更适合。

## 这一阶段适合谁

- 想从软件工程转向 AI / ML 的工程师
- 想补上建模直觉的 LLM 应用开发者
- 已经会调库，但还不清楚模型为何有效、何时失效的初学者

## 学完后你会得到什么

- 能把一个业务需求拆成样本、特征、标签、目标函数与评估指标
- 能判断逻辑回归、决策树、SVM、K-Means、PCA 各自适合什么场景
- 能理解神经网络的前向传播、反向传播与训练循环
- 能区分“模型真的有效”和“只是离线结果看起来漂亮”
- 能更自然地进入后续 CNN、序列模型与 Transformer 的学习

## 本阶段主线案例

### 信用风控：未来 90 天违约预测

假设你在一家消费金融平台工作，系统每天都要决定：

- 哪些用户可以放宽额度
- 哪些用户需要更严格审核
- 哪些用户必须提前预警

如果把高风险用户误判为低风险，平台会直接损失坏账；如果把低风险用户误判为高风险，平台又会损失优质客户和收入。
这正是机器学习与深度学习最适合拿来练基本功的场景：目标具体、代价真实、指标不能乱选。

## 学习地图

| 模块 | 你会掌握什么 | 为什么重要 |
| --- | --- | --- |
| [机器学习基础](machine-learning/README.md) | 监督学习、无监督学习、评估、泛化、特征工程、基线建模 | 先建立“问题定义与方法选择”的能力 |
| [深度学习基础](deep-learning-basics/README.md) | MLP、激活函数、损失函数、反向传播、优化器、训练稳定性 | 理解为什么需要表示学习，以及训练为何会成功或失败 |

推荐顺序：

1. 先完整读完“机器学习基础”，建立建模与评估框架
2. 再进入“深度学习基础”，理解神经网络如何突破线性模型边界
3. 最后进入 Phase 02，学习更具体的网络结构与训练技巧

## 本阶段亮点

- **不是算法罗列，而是建模决策**：每一章都围绕“什么时候该用什么方法”展开。
- **不是空泛公式，而是最小必要推导**：只保留能帮助你理解模型行为的数学。
- **不是玩具例子，而是真实业务代价**：贯穿信用风控案例，讨论误判成本、指标权衡和误差分析。
- **不是只讲原理，而是工程可落地**：数据切分、泄漏、类别不平衡、训练不稳定、排障优先级都会覆盖。

## 进入这一阶段前最容易踩的坑

- 以为模型越复杂越好，忽略简单基线的重要性
- 只看 Accuracy，不看 Recall、AUC 和业务代价
- 把未来信息泄漏进训练数据，离线效果虚高
- 只记结论，不理解“模型为什么会这样学”

## 学习建议

- 把两章当成一个连续故事来读，不要割裂成“传统 ML”和“深度学习”两块知识。
- 看到指标、公式、训练循环时，始终回到主线案例：它们分别在帮平台减少什么风险？
- 如果你未来想做 LLM 工程，这一阶段的价值在于建立判断力，而不是停留在基础模型本身。

## 时间线节点

本模块对应时间线中的以下关键工作：

| 年份 | 工作 | 核心意义 |
|------|------|---------|
| 2012 | ReLU 普及 / GPU 训练 | 激活函数 + 计算基础设施，深度学习训练成为可能 |
| 2012 | ImageNet LSVRC | Benchmark 文化确立，驱动年度竞争 |

| 2016 | A3C / OpenAI Gym | 异步并行 RL + 标准化环境，RL 研究基础设施 |
| 2017 | PPO | 近端策略优化，后来 RLHF 的核心算法 |
| 2017 | AlphaGo Zero | 纯自我对弈，RL 自举能力的极致 |
| 2019 | MoCo（自监督早期） | 无监督视觉表示学习可行性验证 |
| 2020 | SimCLR / AlphaFold 2 | 对比学习成熟 + AI for Science 里程碑 |
