# Monitoring & Observability

[English](README_EN.md) | [中文](README.md)

## Where does this problem come from?

> After a large-model system goes live, "it runs" is just the beginning. Hallucinations, data drift, answer-quality degradation, and falling user satisfaction often don't appear on day one. Monitoring and observability became the lifeline for continuously ensuring system health.

## Learning Objectives

After completing this module, you should be able to answer:
1. What unique metrics should RAG-system observability focus on?
2. How do you detect and respond to data drift and model degradation?
3. What dimensions and benchmarks should an LLM-application evaluation system include?
