# Phase 07: 实战项目

> 把前六个阶段的知识，组装成可以真正交付的系统。
> 工程能力的最终体现不是"会用 API"，而是"能设计、能评估、能维护"。

## 为什么需要实战项目

学完前六个阶段，你拥有了：
- 机器学习的建模直觉（01）
- 神经网络的架构理解（02）
- Transformer 与预训练的系统认知（03）
- LLM 核心的训练、对齐、高效使用（04）
- RAG 与 Agent 的检索增强能力（05）
- MLOps 与生产工程（06）

但这些知识是**分散**的。实战项目的价值在于：在一个真实的系统目标下，**被迫把所有知识连成一条线**。

## 本阶段项目

### 1. [企业级 RAG 系统](enterprise-rag-system/README.md)

构建一个支持百万文档检索的生产级 RAG 系统。

**覆盖技术栈**：
- 文档处理：Chunking 策略、元数据提取
- Embedding：选型与微调
- 向量数据库：Milvus / Weaviate 部署
- 检索增强：Rerank、Query Rewriting、HyDE
- 生成：Prompt 设计、幻觉控制
- 评估：RAGAS 指标体系
- 生产化：缓存、监控、A/B 测试

**对应时间线节点**：2020 RAG 论文 → 2023 LangChain/LlamaIndex → 2024 长上下文

### 2. [自动化微调与部署流水线](finetune-deploy-pipeline/README.md)

从数据准备到上线监控的端到端 LLM 微调工程。

**覆盖技术栈**：
- 数据工程：指令数据构造、质量过滤
- 微调：LoRA / QLoRA 实战
- 对齐：DPO 训练流程
- 评估：基准测试 + 人工评估
- 部署：vLLM 推理服务
- 监控：Weights & Biases、LLM 可观测性

**对应时间线节点**：2021 LoRA → 2022 DPO → 2023 开源生态 → 2024 vLLM
